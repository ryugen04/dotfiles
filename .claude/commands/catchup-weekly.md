# catchup:weekly

直近 7 日分の ~/dev/logs/claude-catchup/dev/*.yaml を読み込み、
以下の "Weekly Knowledge Map" を生成してください。

# WEEKLY KNOWLEDGE MAP

## 1. よく触ったサービス / モジュール TOP5
- 各サービスごとに触った回数や代表的なファイル

## 2. 理解不足が多く現れた領域 TOP3
- confusions や曖昧さが頻出したドメイン/モジュール
- 関連ファイル例と、何が曖昧なのかの要約

## 3. 今週の成長ポイント
- 「自分の言葉で説明できるようになったこと」などを 3〜5 個

## 4. 来週のキャッチアップ計画
- 深掘りテーマを 2〜3 個
- 各テーマごとに
  - 読むべきコード
  - 小さな実装タスク案
  - 目標（何を説明できるようになりたいか）

---

出力スタイルは "learning-forced" です。

必ず次を含めてください：

1. Task Meaning
   - この変更/操作がシステムやドメインの中で持つ意味を 2〜3 文で説明する。

2. Assumption Check
   - この内容を理解するうえで前提となる概念や構造を 3〜5 個挙げる。
   - 特に曖昧だと危険なポイントがあれば明示する。

3. Completion Criteria
   - 「このキャッチアップが完了した」と言える状態を列挙する。
   - その中に必ず「自分の言葉で説明できること」を 1 項目含める。

4. Self-Check Questions
   - 理解を確認するための具体的な質問を 1〜3 問提示する（質問だけで良い）。

トーンはフラットで簡潔な技術文書としてください。
