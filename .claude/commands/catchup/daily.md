# catchup:daily

直近の日付の ~/dev/logs/claude-catchup/dev/YYYY-MM-DD.yaml を読み込み、
以下の "Daily Knowledge Sheet" を生成してください。

# DAILY KNOWLEDGE SHEET

## 1. 今日触ったサービス/モジュール
- service やディレクトリ単位で、ざっくり一覧

## 2. 獲得した知識
### LANGUAGE（言語・ライブラリ）
- 箇条書きで 3〜5 個

### DOMAIN（ビジネス・ドメイン）
- 箇条書きで 3〜5 個

### ARCHITECTURE（設計・責務）
- レイヤー構造や依存関係の理解に関する気づき

## 3. まだ曖昧な領域（To Investigate）
- ログ中の confusions や不明点があれば集約
- 各項目に対して、「放置するとどんなリスクになりそうか」を一言添える
- 危険度を 1〜3 で付与する

## 4. 明日・次回読むべきコード（Next Reading）
- 30 分以内で読めそうな単位で 3〜5 個
- 危険度の高いものに関連するファイルを優先する
- 各項目に「理由」を一文添える

## 5. 理解チェック（3問）
- 今日の作業に基づく具体的な理解確認の質問を 3 つ
- 質問だけでよい（回答はユーザーが考える）

---

出力スタイルは "learning-forced" です。

必ず次を含めてください：

1. Task Meaning
   - この変更/操作がシステムやドメインの中で持つ意味を 2〜3 文で説明する。

2. Assumption Check
   - この内容を理解するうえで前提となる概念や構造を 3〜5 個挙げる。
   - 特に曖昧だと危険なポイントがあれば明示する。

3. Completion Criteria
   - 「このキャッチアップが完了した」と言える状態を列挙する。
   - その中に必ず「自分の言葉で説明できること」を 1 項目含める。

4. Self-Check Questions
   - 理解を確認するための具体的な質問を 1〜3 問提示する（質問だけで良い）。

トーンはフラットで簡潔な技術文書としてください。
