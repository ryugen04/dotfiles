# codex-skill-authoring

Use before creating or editing `SKILL.md`, skill references, scripts, assets, or `agents/openai.yaml`.

## Procedure

1. Read `references/skill-format.md` for the required structure and discovery rules.
2. Read `references/skill-review-checklist.md` before accepting a skill diff.
3. Keep `SKILL.md` concise and procedural.
4. Move detailed tables, schemas, and long examples to `references/`.
5. Do not create README, CHANGELOG, INSTALL, or unrelated docs.
6. Keep reference files one level deep from `SKILL.md`.
7. Validate deterministic scripts with syntax checks or focused tests.
