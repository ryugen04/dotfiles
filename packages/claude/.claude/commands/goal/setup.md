# 目標設定コンテキストのセットアップ

目標設定の土台となる情報を整理し、context.mdを作成する。

## 知識ベース

@skills/knowledge/goal/SKILL.md
@templates/goal/context.md

## 実行手順

### Step 1: 既存コンテキストの確認

```bash
cat .claude/goals/context.md 2>/dev/null || echo "新規作成"
```

既存ファイルがあれば、更新として扱う。

### Step 2: 自分の「こうなりたい」をヒアリング

以下を質問:

1. **短期目標（1年以内）**: 今年中に達成したいこと、身につけたいスキル
2. **中期目標（1〜3年）**: キャリアの方向性、目指すポジション
3. **長期目標（3年以上）**: 将来像、どんなエンジニアになりたいか
4. **価値観**: 仕事で大切にしていること

### Step 3: 会社の期待をヒアリング

以下を質問:

1. **現在のグレード・役割**: 職位、期待される責務
2. **評価基準**: 何をすれば「貢献」「成長」と判断されるか
3. **次のレベルへの要件**: 昇格・昇給に必要なこと
4. **今期特に期待されていること**: 上司から言われていること

### Step 4: 強み・課題の整理

以下を質問:

1. **強み**: 自信があること、得意なこと
2. **伸ばしたい領域**: これから強化したいこと
3. **課題**: 自覚している改善点

### Step 5: 環境の確認

以下を質問:

1. **現在のアサイン**: 担当プロジェクト、チーム
2. **サポート体制**: メンター、相談できる人
3. **制約**: 時間的・組織的な制約

### Step 6: context.md の作成・更新

ヒアリング内容をもとに、テンプレートに沿ってファイルを作成。

## 出力先

```
.claude/goals/context.md
```

## 注意事項

- 全ての質問に答えなくてもOK。わかる範囲で。
- 情報は随時更新可能。完璧を目指さない。
- 上司との1on1前に整理しておくと有用。
