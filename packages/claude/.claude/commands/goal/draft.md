# 個人目標の作成

**期間:** "$ARGUMENTS"

## 知識ベース

@skills/knowledge/goal/SKILL.md
@templates/goal/period-goal.md

## 前提確認

### Step 1: コンテキストの読み込み

```bash
cat .claude/goals/context.md
```

コンテキストが未作成の場合は、先に `/goal:setup` を実行するよう案内。

### Step 2: 期待値の確認

以下を質問:

1. **今期のテーマ**: 特に注力したいこと
2. **会社から言われていること**: 今期特に期待されていること
3. **挑戦したいこと**: 自分として挑戦したいこと

## 目標設計

### Step 3: 3つのアプローチで目標をブレスト

#### 達成型（事業貢献）
- 担当プロジェクトでの成果
- KPI達成への貢献
- 具体的なデリバラブル

#### ストック型（組織への波及）
シニアエンジニアとして特に重要:
- 技術基盤の整備
- ドキュメント・ガイドライン作成
- メンタリング・コードレビュー品質向上
- チーム横断の改善活動
- 採用・オンボーディング貢献

#### 個人スキルUP型
- 新技術の習得
- 知識のアウトプット
- 資格・認定

### Step 4: ウェイト配分の決定

シニアエンジニアの推奨配分:
```
達成型 50〜70% : ストック型 20〜40% : 個人スキルUP型 10%
```

配分を質問し、決定。

### Step 5: 各目標の詳細化

各目標について:
- 概要
- 背景（会社・自分双方のメリット）
- 成功基準（SMART）
- マイルストーン

### Step 6: 品質チェック

以下を確認:
- [ ] 達成したら「レベルアップした」と見なされるか
- [ ] 自分の「なりたい」につながっているか
- [ ] 期内で達成可能な現実的な目標か
- [ ] 成果をアピールしやすい形か

## 出力

テンプレートに沿って目標ファイルを作成。

## 出力先

```
.claude/goals/current-period.md
```

または指定された期間名で:
```
.claude/goals/{期間名}.md
```
