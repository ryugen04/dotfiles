# 月次目標振り返り

**対象月:** "$ARGUMENTS"

## 知識ベース

@skills/goal/SKILL.md
@skills/goal/templates/monthly-review.md

## 実行手順

### Step 1: 現在の目標を読み込み

```bash
cat .claude/goals/current-period.md
```

### Step 2: 先月の振り返りを確認（あれば）

```bash
ls .claude/goals/reviews/
cat .claude/goals/reviews/{前月}.md 2>/dev/null
```

### Step 3: 今月のハイライトをヒアリング

以下を質問:

1. **やったこと（成果）**: 今月達成したこと、完了したタスク
2. **学んだこと**: 気づき、新しく知ったこと
3. **課題**: うまくいかなかったこと、困っていること

### Step 4: 目標別の進捗確認

各目標について:
- 進捗率（%）
- 今月のアクション
- 来月の予定
- 懸念事項

### Step 5: 目標の見直し検討

以下を確認:
- 目標の追加・変更が必要か
- ウェイト配分の調整が必要か
- 環境変化により目標が適切でなくなっていないか

### Step 6: 来月の重点アクション決定

優先度の高い3つのアクションを決定。

### Step 7: 相談事項の整理

上司・メンターに相談すべきことがあれば整理。

## 出力

テンプレートに沿って振り返りファイルを作成。

## 出力先

```
.claude/goals/reviews/{YYYY-MM}.md
```

## 振り返りのコツ

### やったことを「成果」として認識する

> 「意外と、アクションしたことを『成果』と本人が自覚できていないことはよくあります」

小さなことでも記録。期末にアピールできる材料になる。

### 進捗が悪い場合の分析

- アクションが適切でないのか
- サポートが足りないのか
- 環境と目標がマッチしなくなっているのか

### 目標の修正は恥ではない

環境変化に応じて目標を調整するのは健全。
ただし、「なぜ変更したか」は記録しておく。
