# 週次レトロ（/bolt:retro）

1週間分のボルトを振り返り、データ駆動で改善提案を行う。

---

## 実行手順

### 1. 週次データ収集

```bash
# 今週のコミット統計
git log --oneline --since="7 days ago" --format="%h %s" | wc -l
git shortlog --since="7 days ago" -sn

# 日別コミット数
git log --since="7 days ago" --format="%ad" --date=short | sort | uniq -c

# 変更ファイル統計
git diff --stat $(git log --since="7 days ago" --format="%H" | tail -1)^..HEAD 2>/dev/null
```

### 2. 日次ログ読み込み

`.claude/bolt/*.md` から今週分のログを読み込み、パターンを分析。

### 3. 週次サマリ生成

```markdown
## 週次レトロ（YYYY-MM-DD 〜 YYYY-MM-DD）

### サマリ
| 指標 | 今週 | 先週 | 変化 |
|------|-----|-----|------|
| コミット数 | XX | YY | +Z% |
| 完了タスク | XX | YY | +Z% |
| 継続タスク | XX | YY | -Z% |

### 曜日別パターン
| 曜日 | コミット数 | 特記事項 |
|------|----------|---------|
| 月 | X | プランニング日 |
| 火 | X | - |
| 水 | X | リファインメント |
| 木 | X | 実装集中 |
| 金 | X | レトロ |

### 完了タスク一覧
1. タスクA（月）
2. タスクB（火）
3. ...
```

### 4. パターン分析

```markdown
### 発見したパターン

**うまくいったこと**
- 〇〇の手法で効率UP
- △△の分割で見積もり精度向上

**課題**
- □□系のタスクで遅延傾向
- 見積もり超過が多い曜日: X曜

**ボトルネック**
- 〇〇の待ち時間
- △△の調査不足
```

### 5. 改善提案

```markdown
### 改善アクション（次週）

1. **プロセス改善**
   - 〇〇を△△に変更
   - 見積もり時に□□を考慮

2. **ツール活用**
   - Codex CLIに委譲できるタスク: 〇〇系
   - Gemini調査を増やす領域: △△

3. **スキル向上**
   - 〇〇について学習時間を設ける
```

### 6. 週次ログ保存

`.claude/bolt/retro/YYYY-WXX.md` に保存（Wxx = 週番号）。

---

## 出力先

- 週次レトロ: `.claude/bolt/retro/YYYY-WXX.md`

---

## 分析観点

### 定量指標
- コミット数/日
- タスク完了率
- 見積もり精度（計画 vs 実績）

### 定性指標
- よく発生する障害パターン
- 効果的だった手法
- チーム（AI）協業の改善点

---

## Tips

- 金曜夕方に30分で実施
- データに基づく判断（感覚ではなく数値）
- 改善アクションは1-2個に絞る（やりすぎない）
