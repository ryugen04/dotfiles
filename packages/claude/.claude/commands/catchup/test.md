代表的なモジュール/変更を選び、
理解確認のための 5 問のクイズを生成してください。

# UNDERSTANDING CHECK

## Q1〜Q5
- 各質問は、具体的なコードやドメイン挙動に紐づけてください。

## 模範回答
- 各質問に対する模範回答を簡潔に記載してください。

## 読み直すべきファイル候補
- 理解が浅いとバグや設計ミスにつながりそうなファイル/モジュールを 3 件挙げる

---

出力スタイルは "learning-forced" です。

必ず次を含めてください：

1. Task Meaning
   - この変更/操作がシステムやドメインの中で持つ意味を 2〜3 文で説明する。

2. Assumption Check
   - この内容を理解するうえで前提となる概念や構造を 3〜5 個挙げる。
   - 特に曖昧だと危険なポイントがあれば明示する。

3. Completion Criteria
   - 「このキャッチアップが完了した」と言える状態を列挙する。
   - その中に必ず「自分の言葉で説明できること」を 1 項目含める。

4. Self-Check Questions
   - 理解を確認するための具体的な質問を 1〜3 問提示する（質問だけで良い）。

トーンはフラットで簡潔な技術文書としてください。

このコマンドは PR 作成前の理解チェックとしても利用できます。
