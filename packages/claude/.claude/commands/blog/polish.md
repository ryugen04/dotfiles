# ブログ記事 最終校正

**対象:** "$ARGUMENTS"

## ガイドライン

@skills/knowledge/blog/SKILL.md

## 対象ファイル特定

引数が指定されていない場合:
1. 直近に編集された articles/*.md を検索
2. 対象ファイルをユーザーに確認

## チェックリスト

### 1. フロントマター確認

```yaml
---
title: ""      # 空でないか、適切な長さか
emoji: ""      # 記事に合った絵文字か
type: "tech"   # tech or idea
topics: []     # 4つ以下、適切なトピックか
published: false  # 公開可否
---
```

**チェック項目:**
- [ ] titleが空でない
- [ ] titleが65文字以内
- [ ] emojiが設定されている
- [ ] topicsが1-4個
- [ ] topicsがZennで存在するトピック名か

### 2. 必須要素確認

- [ ] **コード例**: 技術記事に最低1つのコードブロックがあるか
- [ ] **バージョン情報**: 使用ツール/言語のバージョンが明記されているか
- [ ] **体験談**: 筆者の経験・感想が含まれているか
- [ ] **参考文献**: 最低1つの外部リンク（公式ドキュメント等）があるか

### 3. 禁止事項の最終確認

**以下がないことを確認:**
- [ ] 見出し・文末のコロン
- [ ] 箇条書きでの太字使用
- [ ] 定型表現（「以下に〜」「次に〜」「最後に〜」）
- [ ] 推測表現（「おそらく」「思われます」）
- [ ] 同一表現の過度な繰り返し

### 4. 読みやすさ確認

- [ ] **段落の長さ**: 1段落が5文を超えていないか
- [ ] **見出しの階層**: h2, h3が適切に使われているか
- [ ] **コードブロック**: 言語指定があるか（```bash など）
- [ ] **リンク**: リンク切れがないか（可能な範囲で）

### 5. Zenn固有の確認

- [ ] **画像**: 画像を使う場合は /images/ に配置されているか
- [ ] **埋め込み**: URL埋め込みが正しく機能するか
- [ ] **長さ**: 極端に短すぎない（500文字以上推奨）
