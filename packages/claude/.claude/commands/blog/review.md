# ブログ記事レビュー

**対象:** "$ARGUMENTS"

## ガイドライン

@skills/knowledge/blog/SKILL.md

## 対象ファイル特定

引数が指定されていない場合:
1. 直近に編集された articles/*.md を検索
2. 対象ファイルをユーザーに確認

## レビュー定義

### 起動するエージェント

1. **blog-style-checker** (agents/blog/)
   - AI臭さ・文体問題の検出
   - コロン使用、箇条書き太字、定型表現、推測表現など

2. **blog-accuracy-checker** (agents/blog/)
   - 技術的正確性の検証
   - バージョン情報、コードエラー、参考文献など

### 実行方法

2つのエージェントを **並列** で起動する（Task tool で parallel指定）。

各エージェントに渡す情報:
- 対象ファイルの全文
- レビュー依頼であること

## 結果フォーマット

```markdown

### 対象ファイル
[ファイルパス]

---

### 文体チェック（blog-style-checker）

#### サマリー
[2-3文の評価]

#### 検出結果
[スコア50以上の指摘を列挙]

#### 総合スコア: XX / 100

---

### 技術的正確性（blog-accuracy-checker）

#### サマリー
[2-3文の評価]

#### 検出結果
[スコア50以上の指摘を列挙]

#### 総合スコア: XX / 100

---

### 総合評価

| 観点 | スコア |
|------|--------|
| 文体 | XX / 100 |
| 正確性 | XX / 100 |
| **総合** | **XX / 100** |

### 推奨アクション（優先度順）
1. [最優先の修正]
2. [次の修正]
3. ...
```

## 注意事項

- 両エージェントの結果を統合して出力
- スコア50以上の指摘のみを含める
- 推奨アクションは両方の結果をマージして優先度順に並べる
- 指摘がない場合は「問題なし」と明記
