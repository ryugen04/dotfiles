# ブログドラフト作成

**引数:** "$ARGUMENTS"

**引数の解析:**
- `--output=<path>` があれば出力先として抽出
- それ以外の部分をトピックとして扱う

## ガイドライン

@skills/knowledge/blog/SKILL.md

## ステップ

### 1. トピック理解

以下を質問で確認する（AskUserQuestionを使用）:

1. **背景・きっかけ**: なぜこの記事を書こうと思ったか
2. **対象読者**: 誰に向けた記事か（初心者/中級者/特定のユースケース）
3. **ゴール**: 読者にどうなってほしいか
4. **スコープ**: 何を含め、何を含めないか

### 2. 構成案の提示

SKILL.mdの記事構成テンプレートに従い、以下を提案:

```markdown
## 構成案

1. はじめに
   - [きっかけ・動機を1-2文で]
2. [問題/課題セクション]
   - [何が問題だったか]
3. [解決方法セクション]
   - [どう解決したか、コード例含む]
4. [応用/Tips（任意）]
5. まとめ
   - [要点を簡潔に]
6. 参考文献
```

ユーザーの確認を得てから次に進む。

### 3. 情報収集

必要に応じて:
- WebSearch: 最新の公式ドキュメント、ベストプラクティス
- WebFetch: 公式ドキュメントの詳細
- Read: 関連するローカルファイル（過去の記事など）

バージョン情報と公式リンクを必ず取得する。

### 4. ドラフト執筆

SKILL.mdのルールに厳密に従い執筆:

**必須チェック（執筆中に常に意識）:**
- コロン禁止（見出し・文末）
- 箇条書きで太字禁止
- 定型表現禁止（「以下に〜」「次に〜」）
- 推測表現禁止（「おそらく」「思われます」）
- 体験談・感想を含める
- バージョン情報を明記
- 参考文献を追加

**Zennフロントマター:**
```yaml
---
title: ""
emoji: ""
type: "tech"
topics: []
published: false
---
```

### 5. 出力

**出力先の決定:**
1. `--output=<path>` が指定されていればそのパスを使用
2. 指定がなければカレントディレクトリの `articles/` を使用

例:
- `/blog:draft KittyのOS別設定 --output=~/dev/projects/public-zenn-docs`
- `/blog:draft KittyのOS別設定` （カレントディレクトリのarticles/に出力）

**ファイル名:**
- 形式: YYYYMMDD-slug.md
- 日付は今日の日付を使用

## 注意事項

- 1回のドラフトで完璧を目指さない
- /blog:review で後からチェック可能
- ユーザーの声（体験・感想）を活かすことを優先
