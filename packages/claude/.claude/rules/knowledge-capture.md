# 知見キャプチャルール（絶対遵守）

NO PROCEEDING TO NEXT TASK WITHOUT DOCUMENTING ERRORS AND LEARNINGS
NO CROSS-AGENT LEARNING WITHOUT CAREFLOW EVIDENCE

## 目的

**同じ試行錯誤を二度と繰り返さない。**

詰まったり、試行錯誤したり、新しい発見があったら、**本来のタスクと並行して**記録を永続化する。
これは実装タスクと同等以上の優先度を持つ。

## トリガー条件

以下のいずれかが発生したら、即座に記録プロセスを開始する:

| トリガー | 例 |
|---------|-----|
| **エラー遭遇** | ビルドエラー、実行時エラー、設定ミス |
| **試行錯誤** | 2回以上の試行が必要だった |
| **ワークアラウンド発見** | 正攻法ではない回避策を使った |
| **新規パターン発見** | 既存ドキュメントにない知見 |
| **ツール制約発見** | MCP/CLI/エージェントの制限 |
| **ワークフロー改善** | 効率的な手順を発見 |

## 記録プロセス（並行実行必須）

### 1. 即座にサブエージェントを起動

**本来のタスクを止めずに、並行して記録エージェントを起動する。**

```
Task → general-purpose
プロンプト: 以下の知見を記録してください...
run_in_background: true
```

### 2. 記録先の選定

| 知見の種類 | 記録先 | 例 |
|-----------|--------|-----|
| **再発防止ルール** | `.claude/rules/{topic}.md` | 破壊的操作禁止、ビルドエラー対処 |
| **作業証跡・検証ログ** | `.careflow/cases/<case_id>/evidence/` | コマンド結果、調査ソース、失敗ログ |
| **委託結果** | `.careflow/cases/<case_id>/results/<order_id>.result.md` | Codex/Claude subagent結果 |
| **ワークフロー手順** | `.claude/docs/guides/workflows/` | Git操作、DB調査手順 |
| **プロジェクト固有知見** | `MEMORY.md` | エージェント使い分け、命名規則 |
| **ドメイン知識** | `.claude/docs/reference/domain/` | ビジネスロジック仕様・業務フロー |
| **サービス障害の対処手順** | `sango.yaml` の該当サービス `runbook:` | 起動エラー、通信エラー、認証エラー |
| **環境チェック項目** | `sango.yaml` の `doctor.checks` | 新しい依存ツール、Docker設定 |
| **セッション内知見** | `.careflow/cases/<case_id>/learnings/` または `.claude/work/learnings/` | `.careflow` を正本、`.claude/work` は一時 |
| **一時的メモ** | `.claude/work/` | 調査中間成果物。最終証跡は `.careflow` に転記 |

### 3. 記録フォーマット

```markdown
## {トピック名}（{日付}追加）

### 状況
何が起きたか、何をしようとしていたか

### 原因
なぜその問題が発生したか

### 解決策
どう対処したか

### 禁止事項（再発防止）
- 今後やってはいけないこと
- やってしまいそうな誤ったアプローチ

### 正しい手順
1. ...
2. ...
```

## 記録の優先度

記録を後回しにすると、絶対に忘れる。

| 状況 | 行動 |
|------|------|
| タスク完了前に詰まった | **並行して**記録エージェント起動 |
| タスク完了時に振り返り | 未記録の知見がないか確認 |
| セッション終了時 | MEMORY.md に要点を追記 |

非自明な作業では、完了前に `EXPECTED_RESULT_PATH` と `.careflow/cases/<case_id>/evidence/` を確認する。

## 既存ルールへのマージ

新しい知見が既存ルールファイルに関連する場合、**新規ファイル作成ではなく既存ファイルに追記**する。

| 関連ルール | ファイル |
|-----------|---------|
| Git操作 | `git-operations.md` |
| DB操作 | `destructive-db-operations.md`, `multi-database.md` |
| スキーマ変更 | `schemafile-integrity.md` |
| ビルド | `build-error-handling.md` |
| サービス障害 | `sango.yaml` の該当サービス `runbook:` / `troubleshoot:` |
| 環境依存 | `sango.yaml` の `doctor.checks` |
| 外部API連携 | 各プロジェクトの `rules/` 該当ファイル |

## 記録しない場合（例外）

- 単純なタイポ修正
- 既にドキュメント化されている内容の再確認
- 一度だけの特殊ケース（再発の可能性がない）

## 合理化対策

| 言い訳 | 反論 |
|--------|------|
| "記録する時間がない" | バックグラウンドエージェントは並行実行。時間は取らない |
| "大した問題じゃない" | 2回目に遭遇した時にそう思えるか? |
| "後で書く" | 後で書かれた試しがない |
| "コードに修正入れたから十分" | コードは「何を」を記録するが「なぜ」は記録しない |

## 違反時の対応

試行錯誤したのに記録しなかった場合:

1. **即座に作業を中断**
2. 記録エージェントを起動
3. 知見を永続化
4. 作業を再開

## Red Flags -- STOP

- エラーが発生したのに記録エージェントを起動していない
- 2回以上リトライしたのに知見を記録していない
- `.careflow` の evidence に検証ログが残っていない
- セッション終了間際で知見が未永続化
