# パッケージインストール禁止（絶対遵守）

NO PACKAGE INSTALLATION WITHOUT EXPLICIT USER APPROVAL

## 原則

**pip install、npm install、brew install、cargo install などのパッケージインストールコマンドを、ユーザーの明示的な承認なしに実行することを絶対に禁止する。**

## 背景

サプライチェーン攻撃が増加している。
悪意のあるパッケージが正規のパッケージ名に似せて公開されていることがある。
一度インストールすると、postinstall スクリプトで任意コードが実行される可能性がある。

## 禁止行為

| 禁止 | 理由 |
|------|------|
| `pip install {package}` を勝手に実行 | サプライチェーン攻撃のリスク |
| `npm install {package}` を勝手に実行 | 同上 |
| `brew install {package}` を勝手に実行 | 同上 |
| `cargo install {package}` を勝手に実行 | 同上 |
| `go install {package}` を勝手に実行 | 同上 |
| その他のパッケージマネージャも同様 | 同上 |

## 正しい手順

1. **まず提案する**: 「〇〇をインストールすると解決できますが、インストールしてよいですか？」
2. **リスク評価を提示する**:
   - パッケージの公開元（GitHub、PyPI など）
   - メンテナンス状況（最終更新日、スター数、コントリビューター数）
   - 既知の脆弱性
3. **ユーザーの明示的な承認を待つ**
4. **承認後に実行する**

## 例外

**例外なし。**

既存の package.json / requirements.txt / Cargo.toml などに記載されている依存関係の `npm install` / `pip install -r` / `cargo build` は許可されるが、**新しいパッケージの追加**は必ずユーザー承認が必要。

## 違反時の対応

もし誤ってインストールコマンドを実行しようとした場合:
1. 即座に中断
2. ユーザーに謝罪
3. 何をインストールしようとしたか報告
