## 収集した知見

### カテゴリ1: Frontmatter プロパティ

#### 1.1 完全なfrontmatterプロパティ一覧

- **情報源**: [Claude Code公式ドキュメント - Skills](https://code.claude.com/docs/en/skills)
- **信頼度**: 高

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | No | スキル名。省略時はディレクトリ名。小文字・数字・ハイフンのみ（最大64文字） |
| `description` | 推奨 | スキルの説明。Claudeが自動ロードの判断に使用 |
| `argument-hint` | No | 補完時のヒント（例: `[issue-number]`） |
| `disable-model-invocation` | No | `true`でClaude自動呼び出しを禁止。手動`/name`のみ |
| `user-invocable` | No | `false`で`/`メニューから非表示 |
| `allowed-tools` | No | 許可するツール。許可制限内でプロンプトなしで実行 |
| `model` | No | 使用するモデル指定 |
| `context` | No | `fork`でサブエージェントコンテキストで実行 |
| `agent` | No | `context: fork`時のサブエージェントタイプ |
| `hooks` | No | スキルライフサイクルにスコープしたフック |

#### 1.2 memoryフィールド（新機能）

- **情報源**: [Qiita - memory frontmatterでサブエージェントに"記憶"を持たせてみた](https://qiita.com/Aochan0604/items/f3f8a6ebc44613717f7c)
- **信頼度**: 中

サブエージェントに「記憶」を持たせるための新しいfrontmatterフィールド。詳細は公式ドキュメントで確認が必要。

#### 1.3 YAML形式のallowed-toolsサポート

- **情報源**: [Zenn - allowed-toolsが便利](https://zenn.dev/kterui9019/articles/001e146d4f0a61)
- **信頼度**: 高

```yaml
---
allowed-tools:
  - Read
  - Grep
  - Glob
  - Bash(git add:*)
  - Bash(git commit:*)
---
```

YAML形式のリストで記述可能になり、可読性が向上。

#### 1.4 context: forkとagentの組み合わせ

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/skills)
- **信頼度**: 高

```yaml
---
name: deep-research
description: Research a topic thoroughly
context: fork
agent: Explore
---
```

`context: fork`でサブエージェントとして実行し、`agent`で実行エージェントタイプを指定。利用可能なエージェント: `Explore`, `Plan`, `general-purpose`、またはカスタムサブエージェント。

#### 1.5 invocation制御の使い分け

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/skills)
- **信頼度**: 高

| 設定 | ユーザー呼び出し | Claude呼び出し | 用途 |
|------|---------------|--------------|------|
| デフォルト | 可 | 可 | 通常のスキル |
| `disable-model-invocation: true` | 可 | 不可 | /commit, /deployなど副作用のある操作 |
| `user-invocable: false` | 不可 | 可 | 背景知識、コンテキスト情報 |

### カテゴリ2: Agent Teams

#### 2.1 Agent Teamsの有効化

- **情報源**: [Claude Code公式ドキュメント - Agent Teams](https://code.claude.com/docs/en/agent-teams)
- **信頼度**: 高

**実験的機能**: デフォルトで無効。settings.jsonで有効化:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

#### 2.2 Agent Teamsのアーキテクチャ

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/agent-teams)
- **信頼度**: 高

| コンポーネント | 役割 |
|--------------|------|
| Team lead | メインセッション。チーム作成・調整 |
| Teammates | 独立したClaude Codeインスタンス |
| Task list | 共有タスクリスト |
| Mailbox | エージェント間メッセージング |

#### 2.3 teammateMode設定

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/agent-teams)
- **信頼度**: 高

```json
{
  "teammateMode": "in-process"  // または "tmux", "auto"
}
```

- `in-process`: すべてのteammateがメインターミナル内で実行
- `tmux`: 各teammateが独自のペインで実行（tmuxまたはiTerm2必要）
- `auto`: tmuxセッション内ならsplit-pane、そうでなければin-process

#### 2.4 Subagents vs Agent Teams

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/agent-teams)
- **信頼度**: 高

| 項目 | Subagents | Agent Teams |
|------|-----------|-------------|
| コンテキスト | 独自。結果はcallerに返る | 独自。完全に独立 |
| コミュニケーション | メインエージェントにのみ報告 | teammate間で直接メッセージ |
| 調整 | メインエージェントが管理 | 共有タスクリストで自己調整 |
| 用途 | 結果のみ重要なフォーカスタスク | 議論・協調が必要な複雑な作業 |
| トークンコスト | 低い | 高い（各teammateが個別インスタンス） |

#### 2.5 Plan承認の要求

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/agent-teams)
- **信頼度**: 高

```
Spawn an architect teammate to refactor the authentication module.
Require plan approval before they make any changes.
```

teammateにplan modeで作業させ、leadが承認するまで実装を開始しない。

### カテゴリ4: その他の新機能

#### 4.1 文字列置換変数

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/skills)
- **信頼度**: 高

| 変数 | 説明 |
|------|------|

| `$N` | `$ARGUMENTS[N]`のショートハンド |
| `${CLAUDE_SESSION_ID}` | 現在のセッションID |

#### 4.2 動的コンテキスト注入

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/skills)
- **信頼度**: 高

```yaml
---
name: pr-summary
description: Summarize changes in a pull request
context: fork
agent: Explore
---

## Pull request context
- PR diff: !`gh pr diff`
- PR comments: !`gh pr view --comments`
```

`` !`command` ``構文でシェルコマンドを実行し、その出力をスキル内容に注入。

#### 4.3 Ultrathink有効化

- **情報源**: [Claude Code公式ドキュメント](https://code.claude.com/docs/en/skills)

## 適用検討項目

### 即座に適用推奨

1. **YAML形式のallowed-tools**: 可読性向上のため既存コマンドを更新
2. **context: fork + agent指定**: 重い処理をサブエージェントに委譲
3. **invocation制御の見直し**: 副作用のあるコマンドに`disable-model-invocation: true`

### 検討が必要

1. **Agent Teams**: 実験的機能。大規模タスクでの検証が必要
2. **動的コンテキスト注入**: 複雑なワークフローでの活用
