# レトロスペクティブ フォーマット

## ファイル名規約

```
.claude/scrum/retro/{YYYYMMDD}-sprint-{N}.md
```

例: `20260129-sprint-3.md`

## テンプレート

```markdown
# Sprint {N} レトロスペクティブ

## 基本情報

- スプリント: {N}
- 期間: {YYYY-MM-DD} 〜 {YYYY-MM-DD}
- 参加者: {名前}
- ファシリテーター: {名前}
- 採用手法: {KPT / 4Ls / Sailboat 等}

## 前回アクションの確認

| アクション | 担当 | 状況 | コメント |
|-----------|------|------|----------|
| {アクション1} | {担当者} | 完了 / 継続 / 未着手 | {補足} |

## 振り返り

### {手法に応じたセクション}

#### Keep（続けること）
- ...

#### Problem（問題点）
- ...

#### Try（試すこと）
- ...

## 改善アクション

| アクション | 担当 | 期限 | 優先度 |
|-----------|------|------|--------|
| {アクション1} | {担当者} | {日付} | 高 / 中 / 低 |

## スプリントの総括

{2-3文でスプリント全体を振り返る}

## チームの健康状態

- エネルギーレベル: ☆☆☆☆☆ (1-5)
- チームワーク: ☆☆☆☆☆ (1-5)
- 達成感: ☆☆☆☆☆ (1-5)
```

## 手法別セクション例

### KPT
```markdown
#### Keep（続けること）
- ...

#### Problem（問題点）
- ...

#### Try（試すこと）
- ...
```

### 4Ls
```markdown
#### Liked（良かったこと）
- ...

#### Learned（学んだこと）
- ...

#### Lacked（不足していたこと）
- ...

#### Longed For（望むこと）
- ...
```

### Start/Stop/Continue
```markdown

- ...

#### Stop（やめること）
- ...

- ...
```

```markdown

- ...

- ...

- ...

- ...
```

## ファシリテーションのコツ

### 手法選択の目安
| 状況 | 推奨手法 |
|------|----------|
| 初回、短時間 | KPT |
| 学びを重視 | 4Ls |
| 行動変容フォーカス | Start/Stop/Continue |
| 感情面を扱いたい | Mad/Sad/Glad |
| 方向性・ビジョン | Sailboat |

### 心理的安全性の確保
- 発言を遮らない
- 批判ではなく改善を
- 「誰が悪い」ではなく「何が問題か」
- 全員が発言できる機会を作る
