# ADR (Architecture Decision Record) フォーマット

## ファイル名規約

```
.claude/docs/adr/{YYYYMMDD}-{日本語タイトル}.md
```

例: `20260129-認証方式の選定.md`

## テンプレート

```markdown
# ADR-{連番}: {タイトル}

## ステータス

提案 / 承認 / 廃止 / 代替（ADR-XXXに置き換え）

## コンテキスト

### 背景
{なぜこの決定が必要か}

### 解決すべき問題
{具体的に何を解決しようとしているか}

### 制約
- {技術的制約}
- {ビジネス上の制約}
- {時間的制約}

## 検討した選択肢

### 選択肢A: {名前}
{概要}

**メリット:**
- ...

**デメリット:**
- ...

### 選択肢B: {名前}
{概要}

**メリット:**
- ...

**デメリット:**
- ...

## 決定

{選択した選択肢とその理由}

## 影響

### Positive
- ...

### Negative
- ...

## 関係者

- 提案者: {名前}
- レビュアー: {名前}
- 承認日: {YYYY-MM-DD}

## 参考資料

- {関連ドキュメント、URL等}
```

## 記載のコツ

### コンテキストで意識すること
- 後から読む人が「なぜこの議論が始まったか」を理解できるように
- 当時の状況を具体的に記述

### 選択肢は偏りなく
- 採用しなかった選択肢も同程度に詳細に記載
- 「なぜ採用しなかったか」が重要

### 影響は正直に
- ネガティブな影響も隠さない
- 将来のトレードオフを明示

### 決定理由は明確に
- 「なんとなく」は禁止
- 根拠を示す
