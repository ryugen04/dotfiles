# 1日ボルト開発

AI駆動開発に最適化された1日単位の開発サイクル。

---

## 従来スプリントとの違い

| 観点 | 従来（2週間） | 1日ボルト |
|------|-------------|-----------|
| 期間 | 2週間 | 1日 |
| プランニング | 3時間以上 | 15分 |
| デイリー | 15分×10回 | 朝計画 + 夕レビュー |
| レトロ | 2週に1回 | 週次（オプション） |
| 実装者 | 人間 | AI + 人間レビュー |

---

## 1日のリズム

```
09:00 ─ ボルト開始（15分）
  └─ /bolt:start
  └─ ゴール設定、タスク選択

09:15 ─ 実装
  ├─ Claude Code: 設計、レビュー
  ├─ Codex CLI: 高速実装
  └─ Gemini CLI: 調査

  （随時）
  └─ /bolt:status で進捗確認

17:00 ─ ボルト終了（15分）
  └─ /bolt:end
  └─ 振り返り、申し送り作成
```

---

## ボルトイベント

### /bolt:start（朝）

1. 昨日の申し送り確認
2. バックログから候補提示
3. 今日のゴール確定
4. タスク登録

**所要時間**: 15分以内
**成果物**: 今日のゴール（TaskList）

### /bolt:status（随時）

1. タスク状況確認
2. Git状況確認
3. 進捗サマリ表示
4. ペース判断（順調/要注意/遅延）

**所要時間**: 5分以内
**推奨頻度**: 1時間ごと

### /bolt:end（夕）

1. 今日の成果収集
2. 差分サマリ生成
3. 振り返り（Keep/Problem/Try）
4. 翌日申し送り作成

**所要時間**: 15分以内
**成果物**: 申し送り、日次ログ

### /bolt:retro（週末）

1. 週次データ収集
2. パターン分析
3. 改善提案

**所要時間**: 30分
**推奨**: 金曜夕方

```
└── retro/
    └── YYYY-WXX.md      # 週次レトロ
```

## ゴール設定の原則

### 良いゴール

- 1日で完了可能な粒度
- 明確な完了条件
- 価値を届ける単位

### 避けるべきゴール

- 「〇〇を進める」（完了条件が曖昧）
- 「調査する」（成果物が不明確）
- 複数日にまたがる大きさ

---

## 遅延時の対応

1. `/bolt:status` で状況確認
2. ゴールの縮小を検討
3. 翌日への持ち越しを決定
4. `/bolt:end` で申し送りに理由を記録

**重要**: 無理に完了させない。持ち越しの理由を記録することで次回の見積もり精度が向上。

---

## 週次レトロの観点

### 定量
- コミット数/日
- タスク完了率
- 見積もり精度

### 定性
- 効果的だった手法
- 発生した障害パターン
- AI協業の改善点
